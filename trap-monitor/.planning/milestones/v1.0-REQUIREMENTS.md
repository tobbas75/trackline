# Requirements: Trap Monitor Hardening

**Defined:** 2026-03-23
**Core Value:** Every concern in CONCERNS.md is resolved — security holes closed, data scoped correctly, firmware edge cases handled, tests covering critical paths, and the dashboard refactored for maintainability.

## v1 Requirements

### Security

- [x] **SEC-01**: Command endpoint rejects requests when DEFAULT_CMD_PIN env var is not set (no fallback to "0000")
- [x] **SEC-02**: Push notify endpoint rejects unauthenticated requests when SUPABASE_WEBHOOK_SECRET is not set (fail closed)
- [x] **SEC-03**: Command endpoint enforces per-IP rate limiting (max 10 requests/minute)
- [x] **SEC-04**: Service role operations are logged with actor identity (user ID or "webhook")
- [x] **SEC-05**: Portal access check fails closed — returns unauthorized when portal.check_app_access() is unavailable

### Data Isolation

- [x] **ISO-01**: Realtime subscriptions filter events by org_id at subscription level (not in-memory)
- [x] **ISO-02**: Event queries are scoped to current org_id before ordering/limiting
- [x] **ISO-03**: Composite index exists on events(org_id, triggered_at) for performant org-scoped queries

### Error Handling

- [x] **ERR-01**: Command log insert failures are caught and logged (not silently swallowed)
- [x] **ERR-02**: GPS coordinates are validated (-90..90, -180..180) before database insertion
- [x] **ERR-03**: Unknown SMS formats trigger a structured alert (not silent UNKNOWN storage)
- [x] **ERR-04**: JSON parsing in dashboard includes schema validation after parse

### Code Quality

- [x] **CQ-01**: BG95 modem stub removed entirely — only Quectel EG800Q driver remains
- [x] **CQ-02**: MapView props typed as Unit[] and TrapEvent[] (no any[])
- [x] **CQ-03**: SMS ingestion removes all any casts — uses typed interfaces for unitUpdate and eventRow
- [x] **CQ-04**: Leaflet icon workaround uses proper type narrowing instead of (as any)
- [x] **CQ-05**: Org deduplication logic replaced with clean database-level query (aggregate or distinct)

### Configuration

- [x] **CFG-01**: Firmware validates config defaults at boot — warns if PIN/phone/unitID are uncustomized
- [x] **CFG-02**: Hardcoded fallback email removed — VAPID_EMAIL is required
- [x] **CFG-03**: Frontend validates all required env vars at startup with clear error messages
- [x] **CFG-04**: DEVICE_TIMEZONE is validated against IANA timezone database in edge function

### Dashboard Refactor

- [x] **DASH-01**: OrgSelector extracted as standalone component with org switching logic
- [x] **DASH-02**: EventList extracted as standalone component with filtering and sorting
- [x] **DASH-03**: UnitGrid extracted as standalone component for unit cards
- [x] **DASH-04**: useDashboardData hook extracted for data fetching and realtime subscriptions
- [x] **DASH-05**: Dashboard page.tsx reduced to composition of extracted components (<200 lines)

### Firmware

- [x] **FW-01**: Retry queue prioritizes ALERT > TRAP > HEALTH messages
- [x] **FW-02**: GPS stale timeout aligned — fix validates using acquisition window, not 2s age check
- [x] **FW-03**: Command listen window has grace period — extends if command received in final 10s
- [x] **FW-04**: Deep sleep entry guarded against concurrent trap interrupt (atomic state check)

### Test Coverage

- [x] **TEST-01**: Vitest configured with TypeScript support for frontend/backend tests
- [x] **TEST-02**: SMS parsing tests cover all 3 formats (TRAP, HEALTH, ALERT) + edge cases
- [x] **TEST-03**: API route tests cover command, orgs, notifications, push endpoints
- [x] **TEST-04**: Key component tests cover dashboard sub-components after refactor
- [x] **TEST-05**: RLS policy tests verify org isolation (pgTAP or integration tests)
- [x] **TEST-06**: Command PIN validation tested (valid, invalid, missing)
- [x] **TEST-07**: Coverage reaches ~70% on critical paths (SMS parsing, API routes, auth)

## v2 Requirements

### Observability

- **OBS-01**: Structured logging with context tags across all tiers
- **OBS-02**: Error aggregation service integration (e.g., Sentry)
- **OBS-03**: Firmware watchdog telemetry (reset reason tracking)

### Advanced Security

- **ASEC-01**: Webhook signature verification (HMAC) for Telstra/Twilio
- **ASEC-02**: Command audit dashboard in UI
- **ASEC-03**: Per-device PIN rotation via SMS command
