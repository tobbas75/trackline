## Phase 6: Firmware Fixes

**Goal:** Fix all 4 firmware edge cases identified in CONCERNS.md.

**Requirements:** FW-01..04

**Plans:** 2/2 plans complete

Plans:
- [x] 06-01-PLAN.md — Priority retry queue (FW-01) and GPS stale timeout fix (FW-02)
- [x] 06-02-PLAN.md — Command listen grace period (FW-03) and deep sleep race guard (FW-04)

**Scope:**
- Implement priority retry queue (ALERT=0 highest, TRAP=1, HEALTH=2)
- Fix GPS stale check — use TinyGPSPlus isUpdated() instead of age() < 2000
- Add command listen window grace period (extend 10s if command received in final 10s)
- Add atomic state check before deep sleep entry (disable interrupts, check trap flag)

**Files likely touched:**
- `firmware/src/main.cpp` (state machine, sleep entry)
- `firmware/src/storage.h` (priority queue)
- `firmware/src/gps.h` (stale fix)
- `firmware/src/config.h` (priority + grace period constants)
- `firmware/src/messages.h` (priority field on EventMessage)

**Success criteria:**
- ALERT messages dequeue before TRAP and HEALTH
- GPS fix accepted when valid within acquisition window
- Command received at T-5s extends listen window by 10s
- Trap interrupt during sleep prep is caught and processed
