# Requirements: Camera Image Pipeline + Dual Ingestion

**Defined:** 2026-03-29
**Core Value:** Unified data pipeline that accepts both SMS and MQTT, with camera image ingestion and multi-species wildlife detection viewing on the dashboard.

## v2 Requirements

### Dual Ingestion Pipeline

- [x] **MQTT-01**: MQTT ingestion edge function receives JSON payloads via HTTP webhook (broker-to-webhook bridge)
- [x] **MQTT-02**: MQTT payload parser handles NE301 JSON format (metadata, device_info, ai_result, image_data)
- [x] **MQTT-03**: Units table has connectivity_method field (sms | mqtt) configurable per device
- [x] **MQTT-04**: MQTT ingestion writes to existing events table for trap events (same schema as SMS events)
- [x] **MQTT-05**: Per-device connectivity switching — changing a unit from SMS to MQTT updates ingestion routing
- [x] **MQTT-06**: MQTT ingestion validates payload structure and rejects malformed data with structured error logging

### Camera Events & Detection Data

- [x] **CAM-01**: camera_events table stores image metadata, device reference, capture timestamp, org_id
- [x] **CAM-02**: camera_detections table stores per-detection data (species, confidence, bounding box x/y/w/h) linked to camera_event
- [x] **CAM-03**: species table (registry) with extensible species list — name, common_name, scientific_name, category
- [x] **CAM-04**: Camera events scoped by org_id with RLS policies matching existing pattern
- [x] **CAM-05**: Camera event ingestion extracts detections from ai_result.detections[] array and stores individually
- [x] **CAM-06**: Device health telemetry stored per camera event (battery_percent, communication_type, inference_time_ms)

### Image Storage

- [x] **IMG-01**: Supabase Storage bucket for camera trap images (org-scoped paths)
- [x] **IMG-02**: Base64 JPEG from MQTT payload decoded and stored in Supabase Storage
- [x] **IMG-03**: camera_events row references stored image URL/path
- [x] **IMG-04**: Storage RLS policies enforce org-level access (users only see their org's images)

### Camera Unit Management

- [x] **UNIT-01**: Units table supports device_type field (trap_monitor | camera_trap)
- [x] **UNIT-02**: Camera units can be created/edited/deleted via existing org unit management UI
- [x] **UNIT-03**: Camera units display device-specific fields (model, firmware_version, last_inference_time)
- [x] **UNIT-04**: Camera unit detail page shows recent camera events with thumbnails

### Dashboard — Image Viewer

- [x] **VIEW-01**: Camera event list shows thumbnail, species detected, confidence, timestamp
- [x] **VIEW-02**: Full image viewer with detection bounding box overlays drawn on image
- [x] **VIEW-03**: Species labels and confidence scores displayed per bounding box
- [x] **VIEW-04**: Filter camera events by species name
- [x] **VIEW-05**: Filter camera events by minimum confidence threshold
- [x] **VIEW-06**: Filter camera events by date range
- [x] **VIEW-07**: Camera events appear in unified timeline alongside trap events (event type indicator)

### Tests

- [x] **TST-01**: MQTT ingestion edge function tested (valid payload, malformed, auth)
- [x] **TST-02**: Camera event + detection storage tested (insert, query, org scoping)
- [x] **TST-03**: Image storage tested (upload, retrieve, org-scoped access)
- [x] **TST-04**: Camera dashboard components tested (event list, image viewer, filters)
- [x] **TST-05**: Coverage maintained at >= 70% after all new code

## Future Requirements

- MQTT broker self-hosting (currently use managed service)
- SMS deprecation pathway (remove SMS ingestion once all devices on MQTT)
- Real-time camera event streaming via Supabase Realtime
- Camera event map view (GPS-tagged detections on Leaflet map)
- Detection confidence analytics / species frequency charts
- Bulk image export (download all images for a date range/species)
- Model performance tracking (accuracy over time per device)

## Out of Scope

| Feature | Reason |
|---------|--------|
| AI model training/deployment | Hardware-side concern, documented in NeoEyes-Addax-Testing/ |
| Camera Trap custom hardware | Separate product, NE301 evaluation only |
| MQTT broker infrastructure | Use managed service (HiveMQ, EMQX, etc.) |
| Removing SMS ingestion | Kept as fallback, not deprecated in this milestone |
| Mobile app | Web dashboard only |
| WildTrack/Fire App changes | Shared tables read-only |
| Video support | Images only in v2, video deferred |
