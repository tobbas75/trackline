import { describe, it, expect } from 'vitest';
import { validateNE301Payload, extractDeviceId, extractTopic, validateTrapPayload, isValidGPS } from './mqtt-validation';

**describe('validateNE301Payload'):**
- `it('accepts a valid NE301 payload')` -- construct a full valid payload with metadata (timestamp, width, height), device_info (battery_percent, communication_type), ai_result with nested ai_result (detection_count, detections array with class_name/confidence/x/y/width/height), and image_data starting with `data:image/jpeg;base64,`. Assert `valid === true`.
- `it('rejects null payload')` -- pass `null`. Assert `valid === false`, reason contains "non-null".
- `it('rejects missing metadata')` -- payload without metadata key. Assert reason contains "metadata".
- `it('rejects non-numeric metadata.timestamp')` -- metadata.timestamp = "not-a-number". Assert reason contains "timestamp".
- `it('rejects zero metadata.width')` -- metadata.width = 0. Assert reason contains "width".
- `it('rejects missing device_info')` -- omit device_info. Assert reason contains "device_info".
- `it('rejects missing ai_result')` -- omit ai_result. Assert reason contains "ai_result".
- `it('rejects missing inner ai_result')` -- ai_result without nested ai_result. Assert reason contains "ai_result.ai_result".
- `it('rejects detection with confidence > 1')` -- detection confidence = 1.5. Assert reason contains "confidence".
- `it('rejects detection with negative x coordinate')` -- detection x = -0.1. Assert reason contains "x".
- `it('rejects missing image_data')` -- omit image_data. Assert reason contains "image_data".
- `it('rejects image_data without base64 prefix')` -- image_data = "not-base64". Assert reason contains "data:image".
- `it('accepts PNG image data')` -- image_data starting with `data:image/png;base64,`. Assert valid === true.

**describe('extractDeviceId'):**
- `it('extracts device_id from URL query param')` -- `new URL('http://x?device_id=DEV001')`, empty body. Assert returns "DEV001".
- `it('extracts device_id from body field')` -- empty URL, body `{ device_id: 'DEV002' }`. Assert "DEV002".
- `it('extracts device_id from MQTT topic')` -- URL with `?topic=ne301/DEV003/upload/report`, empty body. Assert "DEV003".
- `it('extracts device_id from body topic field')` -- body `{ topic: 'ne301/DEV004/upload/report' }`. Assert "DEV004".
- `it('returns null when no device_id available')` -- empty URL, empty body. Assert null.
- `it('prefers direct device_id over topic')` -- URL `?device_id=DIRECT&topic=ne301/TOPIC/x`. Assert "DIRECT".

**describe('extractTopic'):**
- `it('extracts topic from URL query')` -- URL `?topic=ne301/x/upload/report`. Assert returns the topic string.
- `it('extracts topic from body')` -- body `{ topic: 'ne301/y/upload/report' }`. Assert returns body topic.
- `it('returns null when no topic')` -- empty URL, empty body. Assert null.

**describe('validateTrapPayload'):**
- `it('accepts valid trap payload')` -- `{ event_type: 'TRAP', triggered_at: '2024-01-01T00:00:00Z' }`. Assert valid === true.
- `it('accepts HEALTH event type')` -- event_type: 'HEALTH'. Assert valid.
- `it('accepts ALERT event type')` -- event_type: 'ALERT'. Assert valid.
- `it('rejects unknown event type')` -- event_type: 'UNKNOWN'. Assert valid === false, reason contains "event_type".
- `it('rejects missing triggered_at')` -- omit triggered_at. Assert reason contains "triggered_at".
- `it('rejects non-string/number triggered_at')` -- triggered_at: true. Assert reason contains "triggered_at".
- `it('rejects null body')` -- pass null. Assert valid === false.

**describe('isValidGPS'):**
- `it('accepts valid coordinates')` -- isValidGPS(-33.8, 151.2). Assert true.
- `it('rejects lat > 90')` -- isValidGPS(91, 0). Assert false.
- `it('rejects lat < -90')` -- isValidGPS(-91, 0). Assert false.
- `it('rejects lng > 180')` -- isValidGPS(0, 181). Assert false.
- `it('rejects lng < -180')` -- isValidGPS(0, -181). Assert false.
- `it('accepts boundary values')` -- isValidGPS(90, 180). Assert true. isValidGPS(-90, -180). Assert true.

Since there is no dedicated `camera-storage.ts` module in `frontend/src/lib/`, these tests validate the Supabase query patterns used by the ingest-mqtt edge function and dashboard data hooks. The tests mock the Supabase client and verify the expected query chains work correctly.

```typescript
import { describe, it, expect, vi, beforeEach } from 'vitest';
```

**Mock setup pattern** (follow existing pattern from route.test.ts):
```typescript
const mockFrom = vi.fn();
const mockStorage = vi.fn();
const mockSupabase = {
  from: mockFrom,
  storage: { from: mockStorage },
};
```

**describe('Camera event storage operations'):** (TST-02)

- `it('inserts a camera_event row with required fields')`:
  Set up mockFrom to return `{ insert: vi.fn().mockReturnValue({ select: vi.fn().mockReturnValue({ single: vi.fn().mockResolvedValue({ data: { id: 'evt-1' }, error: null }) }) }) }`.
  Call the mock chain: `mockSupabase.from('camera_events').insert({...}).select('id').single()`.
  Assert mockFrom was called with `'camera_events'`.
  Assert the insert received an object with keys: `unit_id, org_id, captured_at, image_width, image_height, model_name, inference_time_ms, battery_percent, communication_type, detection_count`.
  Assert result data.id equals 'evt-1'.

- `it('inserts camera_detections linked to camera_event')`:
  Set up mockFrom for camera_detections insert returning `{ error: null }`.
  Insert array of 2 detections with `camera_event_id: 'evt-1'`, each having `class_name, confidence, x, y, width, height`.
  Assert mockFrom called with `'camera_detections'`.
  Assert insert received array of length 2.
  Assert each detection has camera_event_id = 'evt-1'.

- `it('queries camera_events with detection join for an org')`:
  Set up mockFrom chain: `from('camera_events').select('*, detections:camera_detections(*)').eq('org_id', 'org-1').order('captured_at', { ascending: false })`.
  Mock returns `{ data: [{ id: 'evt-1', detections: [{ class_name: 'possum' }] }], error: null }`.
  Assert query chain was constructed correctly.
  Assert returned data has detections array.

- `it('enforces org scoping -- different org returns empty')`:
  Set up mockFrom chain for org_id = 'org-2' returning `{ data: [], error: null }`.
  Assert result data is empty array.

- `it('handles insert error gracefully')`:
  Set up mockFrom insert returning `{ data: null, error: { message: 'RLS violation' } }`.
  Assert error.message contains 'RLS'.

**describe('Image storage operations'):** (TST-03)

- `it('uploads image to org-scoped path')`:
  Set up `mockStorage('camera-images').upload(path, bytes, { contentType, upsert: false })` returning `{ error: null }`.
  Call with path = `'org-1/DEV001/evt-1.jpg'`, bytes = `new Uint8Array([0xFF, 0xD8])`, contentType = `'image/jpeg'`.
  Assert mockStorage called with `'camera-images'`.
  Assert upload path starts with `'org-1/'`.
  Assert contentType is `'image/jpeg'`.

- `it('retrieves signed URL for org-scoped image')`:
  Set up `mockStorage('camera-images').createSignedUrl('org-1/DEV001/evt-1.jpg', 3600)` returning `{ data: { signedUrl: 'https://...' }, error: null }`.
  Assert signedUrl is returned.

- `it('rejects upload to different org path')`:
  Set up upload returning `{ error: { message: 'RLS: insufficient permissions' } }`.
  Assert error is returned.

- `it('handles image retrieval for non-existent path')`:
  Set up createSignedUrl returning `{ data: null, error: { message: 'Object not found' } }`.
  Assert error is returned.

- `it('uploads PNG images with correct content type')`:
  Set up upload with contentType = 'image/png', path ending in '.png'.
  Assert upload called with correct contentType.

Each test should build the mock chain fresh in the test body (not shared mutable state) to keep tests isolated. Use `beforeEach(() => { vi.clearAllMocks(); })`.
