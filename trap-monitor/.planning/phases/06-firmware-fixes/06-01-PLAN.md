<task type="auto">
  <name>Task 2: Implement priority-aware queue operations and fix GPS stale validation</name>
  <files>firmware/src/storage.h, firmware/src/gps.h</files>

  **storage.h changes (per FW-01 -- priority queue):**

  The approach: keep append-at-end enqueue but change dequeue and flush to scan for highest priority. This avoids ArduinoJson array insertion complexity on embedded.

  1. **storageSerializeMessage()** -- Add priority to JSON serialization. Add after existing fields:
     ```cpp
     entry["pri"] = msg.priority;
     ```

  2. **storageDeserializeMessage()** -- Add priority deserialization with fallback for legacy entries. Use ArduinoJson's `|` (or) operator:
     ```cpp
     msg.priority = entry["pri"] | msgPriorityFromType(msg.type);
     ```
     This handles queue entries persisted before the priority field existed by deriving priority from the type string.

  3. **storageEnqueue() overflow eviction** -- Replace the current `while (queue.size() >= MAX_QUEUE_SIZE) { queue.remove(0); }` FIFO eviction with priority-aware eviction. When the queue is full, find and remove the lowest-priority entry (highest `pri` value). If multiple entries share the lowest priority, evict the oldest (lowest index):

     ```cpp
     while (queue.size() >= MAX_QUEUE_SIZE)
     {
       int evictIdx = 0;
       int lowestPri = queue[0]["pri"] | MSG_PRIORITY_HEALTH;
       for (int i = 1; i < (int)queue.size(); i++)
       {
         int p = queue[i]["pri"] | MSG_PRIORITY_HEALTH;
         if (p > lowestPri)
         {
           lowestPri = p;
           evictIdx = i;
         }
         // If tied, keep evictIdx as-is (older = lower index = evict first among ties)
       }
       Serial.printf("[QUEUE] Overflow -- evicting idx=%d pri=%d\n", evictIdx, lowestPri);
       queue.remove(evictIdx);
     }
     ```

  4. **storageDequeueNext()** -- Replace taking `queue[0]` with a priority scan. Find the highest-priority entry (lowest `pri` value). If tied, take the oldest (lowest index):

     ```cpp
     int bestIdx = 0;
     int bestPri = queue[0]["pri"] | MSG_PRIORITY_HEALTH;
     for (int i = 1; i < (int)queue.size(); i++)
     {
       int p = queue[i]["pri"] | MSG_PRIORITY_HEALTH;
       if (p < bestPri)
       {
         bestPri = p;
         bestIdx = i;
       }
       // If tied, keep bestIdx (lower index = older = dequeue first)
     }
     JsonObjectConst entry = queue[bestIdx].as<JsonObjectConst>();
     msg = storageDeserializeMessage(entry);
     queue.remove(bestIdx);
     ```

     Add a MARKER log after deserialization:
     ```cpp
     Serial.printf("[MARKER][QUEUE] Dequeued type=%s pri=%d retries=%d\n",
                   msg.type.c_str(), msg.priority, msg.retryAttempts);
     ```

  5. **storageFlushQueue()** -- Change from sequential `queue[0]` processing to priority-order. Replace the `while (queue.size() > 0)` body:
     - Find highest-priority entry using the same scan pattern as dequeueNext
     - Deserialize that entry, format SMS, attempt send
     - On success: remove the entry, increment sentCount, delay(2000) for throttle
     - On failure: increment retryAttempts, check against retryLimit. If exceeded, remove and log drop. If not exceeded, update the entry in-place (clear JsonObject, re-serialize). Log failure and break (stop flush).

     Add MARKER log for flush failures:
     ```cpp
     Serial.printf("[MARKER][QUEUE] Flush fail type=%s pri=%d retries=%d/%d\n",
                   msg.type.c_str(), msg.priority, msg.retryAttempts, retryLimit);
     ```

  **gps.h changes (per FW-02 -- fix stale timeout):**

  Replace the GPS validation condition. The current check `_gps.location.age() < 2000` rejects valid fixes during cold start when inter-NMEA-sentence gaps exceed 2 seconds (common with u-blox M10 during first fix).

  Replace:
  ```cpp
  if (_gps.location.isValid() && _gps.location.age() < 2000) {
  ```

  With:
  ```cpp
  if (_gps.location.isValid() && _gps.location.isUpdated()) {
  ```

  `isUpdated()` returns true if the location was decoded from the most recent NMEA sentence, which precisely captures "we just got a new fix" without an arbitrary time window. Since we are in an active serial read loop, this will be true on the iteration immediately after a GGA/RMC sentence containing coordinates is decoded.

  Add an explanatory comment above the condition:
  ```cpp
  // isUpdated() = location was decoded in the latest NMEA sentence.
  // Previous age() < 2000 check rejected valid fixes during cold-start
  // inter-sentence gaps (FW-02 fix).
  ```
