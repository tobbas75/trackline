    **Mocking strategy:** checkAppAccess and getUserApps accept a `SupabaseClient` parameter directly (they do NOT call createClient internally). This means we create a mock object inline — no module-level vi.mock needed.

    ```typescript
    import { describe, it, expect, vi } from 'vitest';
    import { checkAppAccess, getUserApps } from './check-access';

    function createMockSupabase(overrides: { rpcResult?: any; schemaResult?: any } = {}) {
      return {
        rpc: vi.fn().mockResolvedValue(overrides.rpcResult ?? { data: [], error: null }),
        schema: vi.fn().mockReturnValue({
          from: vi.fn().mockReturnValue({
            select: vi.fn().mockResolvedValue(overrides.schemaResult ?? { data: [], error: null }),
          }),
        }),
      } as any;
    }
    ```

    - Test success: `rpc` returns `{ data: [{ has_access: true, user_role: 'admin' }], error: null }` -> verify hasAccess=true, role="admin"

    - Test empty data: `rpc` returns `{ data: [], error: null }` -> verify hasAccess=false
