<objective>
Scope the cards dashboard view's data access -- realtime subscriptions and event queries -- to the user's current organization, completing data isolation across all dashboard views.

Purpose: The main dashboard hook (`useDashboardData.ts`) and the database layer (migration 011 adding `org_id` to events, composite indexes in migrations 009 and 011) are already correctly implemented. However, the cards view (`cards/page.tsx`) still has the original unscoped pattern: its event query fetches ALL events without org filtering, and its realtime subscriptions have no server-side filters. This means on the cards view: (a) the client receives events for other orgs (wasted bandwidth, potential data leak if RLS changes), (b) the event query's "latest 100" may include events from other orgs that RLS filters out after the limit is applied, returning fewer than 100 visible results, and (c) unit INSERT/UPDATE realtime events from other orgs are received unnecessarily.

Output: Modified cards page with org-scoped queries and subscriptions matching the pattern already established in useDashboardData.ts.
</objective>
