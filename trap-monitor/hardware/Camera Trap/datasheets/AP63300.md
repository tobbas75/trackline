## Feedback Resistor Configuration

**CRITICAL: Verify VFB reference voltage from datasheet before calculating resistors.**

Assuming VFB = 0.6V (common for Diodes Inc. bucks):

```
Vout = VFB x (1 + R1/R2) = 0.6V x (1 + R1/R2)
```

If VFB = 0.765V (also seen in some Diodes Inc. parts):

```
Vout = 0.765V x (1 + R1/R2)
```

### Feedback Resistor Values for This Design

**The following values assume VFB = 0.6V. RECALCULATE if VFB is different.**

#### U11: 3.3V Output (Main Digital Rail, Always On)

```
3.3V = 0.6V x (1 + R1/R2)
R1/R2 = 4.5
```

Suggested: R1 = 180k, R2 = 40.2k (ratio = 4.478, Vout = 3.287V)
Or: R1 = 270k, R2 = 60.4k (same ratio, higher impedance)

#### U13: 3.8V Output (Modem VBAT Rail, PB7 Gated)

```
3.8V = 0.6V x (1 + R1/R2)
R1/R2 = 5.333
```

Suggested: R1 = 160k, R2 = 30.1k (ratio = 5.316, Vout = 3.79V)
Or: R1 = 320k, R2 = 60.4k

#### U15: 5.0V Output (LED VLED Rail, PB6 Gated)

```
5.0V = 0.6V x (1 + R1/R2)
R1/R2 = 7.333
```

Suggested: R1 = 220k, R2 = 30.1k (ratio = 7.309, Vout = 4.985V)
Or: R1 = 440k, R2 = 60.4k

**IMPORTANT: All resistor values above are estimates using standard E96 values. Verify VFB from the datasheet, then recalculate with exact E96 values. TI/Diodes datasheets typically include a recommended resistor table.**

**If VFB = 0.765V:**
- 3.3V: R1/R2 = 3.314 --> R1 = 100k, R2 = 30.1k
- 3.8V: R1/R2 = 3.967 --> R1 = 120k, R2 = 30.1k
- 5.0V: R1/R2 = 5.536 --> R1 = 165k, R2 = 30.1k (non-standard — use 169k or adjust R2)

### Thermal Analysis — U13 Modem Buck (Worst Case)

During LTE TX burst:
- Vin = 8.4V (full battery, worst case for thermal)
- Vout = 3.8V
- Iout = 1A (modem TX)
- Duration = ~5 seconds

Power dissipated in converter = Pout x (1/eta - 1) = 3.8W x (1/0.90 - 1) = 0.42W

With theta-JA ~150 C/W: deltaT = 0.42 x 150 = 63C
At 52C ambient: Tj = 52 + 63 = 115C

**This is marginal.** The burst is only 5 seconds (thermal time constant of SOT-23-6 is ~10-20 seconds), so steady-state is not reached. But thermal pad and copper pour under U13 is CRITICAL.

**Recommendation:** Provide generous GND copper pour under and around U13. Consider 4-layer PCB with thermal vias to inner ground plane. If prototype shows thermal issues, add copper area or consider the larger AP63356 (DFN-8) package.

## Application Notes for This Design

### U11 — 3.3V Always-On Rail

- **Input:** Battery direct (6.0-8.4V after Schottky diodes)
- **Output:** 3.3V, powers STM32U0, PIR sensors, NOR flash, GPS (when enabled), SD card, IMX678 AVDD (via ferrite filter), and feeds second-stage regulators (TPS62088 x2, TPS7A02)
- **EN pin:** Tied to VIN (always enabled) or to VIN via resistor divider for UVLO
- **Iq = 22 uA** — This dominates the sleep budget (44% of 50uA total). See POWER_SYSTEM_REVIEW.md for Phase 2 mitigation strategy (sleep-only LDO bypass).
- **Load range:** ~50uA (sleep) to ~2A (all subsystems active)

### U13 — 3.8V Modem VBAT Rail

- **Input:** Battery direct
- **Output:** 3.8V to modem daughter card (Quectel EG800Q-EU VBAT)
- **EN pin:** Controlled by STM32N6 PB7 (MODEM_3V8_EN)
  - PB7 HIGH = buck enabled, modem powered
  - PB7 LOW = buck shutdown (1uA), modem completely off
- **CRITICAL:** The modem's own power-off mode draws ~55uA. The EN pin MUST be driven LOW to achieve 1uA shutdown. Relying on modem sleep modes is insufficient.
- **OVP:** TVS diode (SMBJ4.5A) on modem card clamps output to protect modem (abs max 4.5V)
- **Peak load:** 1A during TX burst (2A absolute peak, <10ms)
- **Bulk capacitors on modem card:** 100uF electrolytic + 10uF ceramic + 100nF ceramic

### U15 — 5.0V LED VLED Rail

- **Input:** Battery direct
- **Output:** 5.0V to LED daughter board via JST GH connector
- **EN pin:** Controlled by STM32N6 PB6 (LED_5V_EN)
  - PB6 HIGH = buck enabled, LED driver powered
  - PB6 LOW = buck shutdown (1uA), LEDs completely off
- **Load:** 1.4A during IR strobe (8x LEDs at 175mA each through AL8861 buck driver)
- **Duty cycle:** Strobed 300ms per trigger (10 frames x 30ms), 35 night triggers/day = 10.5 seconds/day of LED power
- **Thermal:** Not a concern — very low duty cycle

### PCB Layout Guidelines

1. **Input capacitor** as close as possible to VIN and GND pins
2. **SW trace** short and wide to inductor — this is the high-current switching node
3. **GND plane** continuous under the converter — no splits
4. **Feedback resistors** close to FB pin, route away from SW node
5. **BST capacitor** close to BST and SW pins
6. **Thermal pad:** SOT-23-6 relies on PCB copper for heat dissipation. Provide generous copper pour on GND pins connected to inner ground plane via thermal vias
7. **Keep high-impedance FB trace** away from SW node and inductor to prevent noise coupling

## AP63300 vs AP63301

The AP63301 is the same device but with the EN pin internally tied to VIN (always on). For this design, we need the EN pin for power gating on U13 and U15, so **AP63300 is required** for all three instances (U11 also benefits from having EN available for UVLO or controlled startup).

## Known Issues / Errata

1. **22uA Iq dominates sleep budget.** This is the single largest power consumer in sleep mode (44% of total). The AP63300 was chosen because it can handle the 6.0-8.4V battery input — lower-Iq alternatives (TPS62A01 at ~15uA) cannot survive the battery voltage. See POWER_SYSTEM_REVIEW.md section 4 for Phase 2 mitigation.

2. **SOT-23-6 thermal limitations.** At 1A continuous from 8.4V to 3.8V, the package runs hot. The modem TX burst is only 5 seconds, so it doesn't reach steady state, but adequate copper pour is essential. The LED burst (1.4A, 300ms) is too short to cause thermal issues.

3. **Output ripple.** At 500kHz switching, output ripple is higher than the TPS62088 (4MHz). Typical: 20-30mV peak-to-peak. The 3.3V rail feeds the IMX678 AVDD via a ferrite + LC filter to reduce ripple to acceptable levels for the image sensor analog supply.

4. **VFB reference voltage uncertainty.** Training data shows both 0.6V and 0.765V for various Diodes Inc. buck converters. The correct VFB for the AP63300 MUST be verified from the datasheet before calculating feedback resistors. Using the wrong reference voltage will produce the wrong output voltage.

5. **Minimum input voltage.** The 3.8V minimum is the specification. At 6.0V (empty 6S2P NiMH), there is 2.2V headroom above minimum — comfortable. However, with Phase 1 Schottky diodes (0.3V drop), the effective minimum is 5.7V — still 1.9V above minimum.

6. **EN pin leakage.** When EN is driven by a GPIO through a long PCB trace, verify the EN pin's input leakage current doesn't cause issues. A 100k pull-down resistor on EN ensures clean shutdown if the GPIO tristates during MCU reset.
