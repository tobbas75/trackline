## Feedback Resistor Configuration

The output voltage is set by an external resistor divider from VOUT to FB to GND:

```
Vout = VFB x (1 + R1/R2) = 0.6V x (1 + R1/R2)
```

Where:
- VFB = 0.6V (internal reference)
- R1 = top resistor (VOUT to FB)
- R2 = bottom resistor (FB to GND)

### Recommended Resistor Values for This Design

**U10a: VDDCORE 0.81V (VOS nominal, 600 MHz)**

```
0.81V = 0.6V x (1 + R1/R2)
R1/R2 = 0.35
```

### CONFIRMED from DK Board Schematic (mb1939-n6570-c02, Sheet 5 "External SMPS")

**TPS62088 VFB = 0.600V** (typ; 594mV min / 606mV max — TI datasheet Section 6.5 Page 5)

**Resistor network (read directly from schematic):**

| Ref | Value | Function |
|-----|-------|----------|
| R38 | **56 kΩ** | Upper: VDDCORE → FB (always active) |
| R29 | **160 kΩ** | Lower: FB → GND (always active) |
| R157 | **422 kΩ** | Lower parallel path via Q7 (switched by PF4) |
| Q7 | **Si1062X** | N-MOSFET, gate driven by PF4 |
| R223 | **1 MΩ** | Gate bias / pulldown on Q7 |

**Voltage switching mechanism:**
- **PF4 = LOW** → Q7 OFF → lower resistor = R29 alone (160 kΩ)
  - VDDCORE = 0.6V × (1 + 56k/160k) = **0.810V** (VOS nominal, 600MHz)
- **PF4 = HIGH** → Q7 ON → lower resistor = R29 ‖ R157 = 160k ‖ 422k = **115.9 kΩ**
  - VDDCORE = 0.6V × (1 + 56k/115.9k) = **0.890V** (VOS overdrive, 800MHz)

> Schematic annotation confirms: "PWR_LP = 1 (OVERDRIVE): VDDCORE = 0.890V / PWR_LP = 0 (NOMINAL): VDDCORE = 0.810V"

**Copy this circuit exactly for U10a.** Use the same resistor values and FET (Si1062X or equivalent small-signal N-FET). R223 (1MΩ) ensures PF4 defaults LOW on reset → 0.81V startup → safe nominal mode.

**U10b: IMX678 DVDD 1.1V (fixed)**

```
1.1V = 0.6V x (1 + R1/R2)
R1/R2 = 0.833
```

Suggested: R1 = 100k, R2 = 120k (ratio = 0.833, Vout = 1.1V)
Or: R1 = 200k, R2 = 240k (higher impedance, less divider current)

**IMPORTANT: These resistor values are calculated from the formula. Verify against TI's resistor selection table in the datasheet, which accounts for feedback current and preferred E96 values.**

## Power — Efficiency Considerations

### U10a (3.3V in, 0.81V out, VDDCORE)

At 3.3V input to 0.81V output:
- Duty cycle: ~24.5% (0.81/3.3)
- Active load (NPU running): ~300-450mA — efficiency likely 85-90%
- Sleep load (VBAT retention): ~8uA — PFM mode, efficiency drops but absolute loss is tiny
- 4MHz switching means small inductor and capacitors

### U10b (3.3V in, 1.1V out, IMX678 DVDD)

At 3.3V input to 1.1V output:
- Duty cycle: ~33.3%
- Active load (sensor capturing): ~100-200mA — efficiency likely 88-92%
- Idle: sensor powered down, load near zero

### Input/Output Capacitor Recommendations

Likely requirements (VERIFY from datasheet):
- Input: 10uF ceramic (MLCC), 0402 or 0603, close to VIN pins
- Output: 10-22uF ceramic, low ESR, close to inductor/output
- Inductor: 0.47-1.0uH at 4MHz switching, saturation current > 3A

**At 4MHz switching frequency, the inductor can be very small — this is a key advantage of the TPS62088 for space-constrained designs.**

## PCB Layout — CRITICAL for DSBGA

The TPS62088 in DSBGA-6 is only 1.2 x 0.8 mm — one of the smallest power converters available. PCB layout is critical:

1. **Pad design:** Follow TI's recommended land pattern exactly. DSBGA bumps are ~0.3mm diameter at 0.4mm pitch. NSMD (non-solder-mask-defined) pads recommended.

2. **Via-in-pad:** May be required for GND connection. If used, must be filled and planarised to prevent solder wicking.

3. **Inductor placement:** As close as physically possible to SW bump. Short, wide trace for the SW node to minimise ringing and EMI.

4. **Input capacitor:** Place directly adjacent to VIN bumps. Short loop to GND.

5. **Output capacitor:** Place close to load. Low-ESR ceramic.

6. **Feedback resistors:** Place close to FB bump. Route FB trace away from SW node to avoid noise coupling.

7. **Thermal:** Despite the tiny package, 3A output means significant power dissipation. Ensure adequate copper pour on GND plane under the device. Four-layer PCB recommended.

8. **Assembly:** Requires fine-pitch reflow capability. X-ray inspection recommended to verify solder joints — visual inspection is not possible on BGA.

9. **Stencil:** 0.1mm thick stencil with aperture reduction (50-80% of pad area) to prevent bridging.

**This package is NOT hand-solderable. Do not attempt rework without BGA rework station.**

## Application Notes for This Design

### U10a — VDDCORE with Voltage Switching

This circuit is copied from the STM32N6570-DK development board. The DK board uses:
- TPS62088 with adjustable output
- Two-state feedback divider controlled by STM32N6 GPIO PF4
- LOW = 0.81V (VOS nominal, CPU at 600MHz)
- HIGH = 0.89V (VOS overdrive, CPU at 800MHz, full NPU speed)

The switching is implemented with a small-signal FET (likely N-channel, similar to BSS138) that parallels an additional resistor across the bottom feedback resistor when turned on, changing the divider ratio.

**Switching transient:** When PF4 changes state, the output voltage transitions over the converter's bandwidth (~100-200us). The STM32N6 firmware must wait for the voltage to settle before changing clock speed. The power-good output may toggle during the transition.

**Power-up sequence:** The regulator starts at the default divider ratio (PF4 defaults LOW after reset = 0.81V). Firmware can switch to overdrive after boot if needed for AI inference.

### U10b — IMX678 DVDD (Fixed 1.1V)

Simpler circuit — fixed feedback divider. No switching required.

**Power sequencing requirement:** IMX678 requires DVDD (1.1V) to be present before DOVDD (1.8V) and AVDD (3.3V). Sequence: U10b enable first, then U12 (TPS7A02 1.8V), then AVDD filter (3.3V). All within 200ms.

### Input Rail

Both TPS62088 instances are fed from the 3.3V rail (AP63300 U11 output). This is safe:
- 3.3V is within the 2.4-5.5V input range
- The 3.3V rail is always on (AP63300 Iq = 22uA)
- Cascaded efficiency: ~90% (AP63300) x ~88% (TPS62088) = ~79% from battery to VDDCORE

## Known Issues / Errata

1. **No WSON-6 variant exists.** The "TPS62088DSGR" part number that sometimes appears in reference designs or AI-generated BOMs is WRONG. Only DSBGA (YFP) and PowerWCSP (YWC) packages are available.

2. **DSBGA assembly difficulty.** This is a wafer-level BGA at 1.2x0.8mm — near the limit of standard SMT assembly. Verify your PCB assembler (e.g., JLCPCB) can handle 0.4mm pitch BGA before committing to this part. Request X-ray inspection.

3. **Feedback divider noise sensitivity.** At 0.6V reference with high-value resistors, noise on the FB pin can cause jitter on the output. Keep FB trace short and away from the SW node. Consider adding a small capacitor (10-22pF) across R1 for noise filtering, but verify stability — this adds a zero to the control loop.

4. **Output ripple at light load.** In PFM mode (light load/sleep), the converter operates in pulse-skipping mode. Output ripple increases compared to forced-PWM mode. For VDDCORE this is acceptable. For IMX678 DVDD, verify ripple spec against IMX678 requirements.

5. **Minimum load.** No minimum load required — the converter handles zero load gracefully in PFM mode.
