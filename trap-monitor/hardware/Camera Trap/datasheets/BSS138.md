## Application 1: I2C Bidirectional Level Shifter (NXP AN10441 Topology)

### Circuit — 1.8V to 3.3V (IMX678 Sensor I2C)

This is the standard NXP AN10441 bidirectional level shifter used throughout both the Camera Trap and Trap Monitor designs.

```
1.8V Rail (DOVDD)              3.3V Rail (VDD)
    │                              │
   [Rp_LV]                       [Rp_HV]
    │                              │
    ├──── Source (pin 2)   Drain (pin 3) ────┤
    │         │    BSS138    │               │
    │         └──── Gate (pin 1) ────┘       │
    │              │                         │
    │           1.8V Rail                    │
    │                                        │
  SDA_LV                                  SDA_HV
(1.8V side)                             (3.3V side)
```

Where:
- Rp_LV = pull-up resistor on low-voltage side (to 1.8V)
- Rp_HV = pull-up resistor on high-voltage side (to 3.3V)
- Gate tied to low-voltage rail (1.8V)

### How It Works

1. **Idle state:** Both sides pulled HIGH by pull-ups. BSS138 Vgs = 0V (gate at 1.8V, source pulled to 1.8V by Rp_LV). FET is OFF. Both sides at their respective rail voltages.

2. **Low-side device pulls LOW:** SDA_LV pulled to 0V. Source goes to 0V. Vgs = 1.8V - 0V = 1.8V > Vth. FET turns ON. Drain pulled LOW through FET. SDA_HV goes LOW. Signal transfers from 1.8V side to 3.3V side.

3. **High-side device pulls LOW:** SDA_HV pulled to 0V. Drain goes to 0V. Body diode (source to drain) conducts, pulling source (SDA_LV) to ~0V + Vf. Actually, since drain is lower than source, current flows through the body diode, pulling source low. Then Vgs = 1.8V - 0V = 1.8V. FET turns on, clamping SDA_LV to near 0V. Signal transfers from 3.3V side to 1.8V side.

4. **Release:** When the pulling device releases, pull-ups restore both sides to HIGH.

### Pull-Up Resistor Selection

| Parameter | Value | Notes |
|-----------|-------|-------|
| Rp_LV (1.8V side) | 4.7k ohm | For I2C Standard/Fast mode |
| Rp_HV (3.3V side) | 4.7k ohm | Matched to low side |

**BOM reference:** R_SDA_33, R_SCL_33 = 4.7k ohm, 0402 (3.3V side pull-ups on main board). Sensor-side pull-ups to 1.8V are on the sensor's local decoupling area.

### I2C Speed Limitations at 1.8V Gate Drive

The BSS138 gate threshold (Vgs(th)) is 0.5V to 1.5V. At 1.8V gate drive:
- **Worst case (Vgs(th) = 1.5V):** Only 0.3V overdrive. Rds(on) is HIGH (~30+ ohm). Turn-on is slow.
- **Typical (Vgs(th) = 0.8V):** 1.0V overdrive. Rds(on) ~10-20 ohm. Adequate for I2C.
- **Best case (Vgs(th) = 0.5V):** 1.3V overdrive. Rds(on) ~5-10 ohm. Good.

**Impact on I2C speed:**

The RC time constant of the level shifter limits the maximum I2C clock frequency:
- Rise time: determined by pull-up resistor charging bus capacitance (Rp x Cbus)
- Fall time: determined by Rds(on) plus driver impedance discharging Cbus
- BSS138 Ciss (30pF) adds to the bus capacitance on both sides

For I2C Standard Mode (100 kHz): Easily achievable with 4.7k pull-ups and BSS138 at 1.8V gate.

For I2C Fast Mode (400 kHz): Likely achievable but marginal. The ~30pF Ciss of the BSS138 adds to the total bus capacitance. With short traces (<10mm, as specified in BOM) and 4.7k pull-ups:
- Total bus capacitance per side: ~30pF (BSS138) + ~10pF (traces) + ~5pF (device input) = ~45pF
- Rise time: Rp x Cbus = 4.7k x 45pF = 212ns
- I2C Fast Mode requires tr < 300ns (max) — passes, but no margin
- Reducing pull-ups to 2.2k improves: 2.2k x 45pF = 99ns — comfortable

For I2C Fast Mode Plus (1 MHz): NOT recommended with BSS138 at 1.8V gate. Use a dedicated level shifter IC or reduce pull-ups to 1k ohm (increases standby current).

**Recommendation for this design:** Use 4.7k pull-ups for I2C Standard Mode (100 kHz) to the IMX678. The sensor's register access is not speed-critical — it happens once at initialization and occasionally for AE/gain readback. If faster access is needed, reduce pull-ups to 2.2k and verify signal integrity on the prototype.

### Placement

BOM notes specify: "Place within 10mm of IMX678." This minimizes trace capacitance and ensures clean signal transitions.

## Application 2: Gate Inverter for P-FET Power Switches

### Circuit (Q4 — GPS Power Gate Inverter)

```
3.3V Rail
    │
   [10k pull-up]
    │
    ├──── SI2301 Gate
    │
    ├──── BSS138 Drain (pin 3)
    │
    │     BSS138 Gate (pin 1) ◄── GPIO PB5
    │
    └──── BSS138 Source (pin 2) ── GND
```

**Operation:**
- PB5 HIGH (3.3V): BSS138 Vgs = 3.3V, turns ON hard (Rds ~1.5 ohm). Drain pulled to ~0V. SI2301 gate = ~0V, Vgs = -3.3V. P-FET ON. Load powered.
- PB5 LOW (0V): BSS138 OFF. 10k pull-up pulls SI2301 gate to 3.3V. P-FET OFF. Load unpowered.

At Vgs = 3.3V, the BSS138 is well into full enhancement (Rds(on) ~2-3 ohm). The current through the gate inverter is negligible (only the SI2301 gate charge, ~8.5nC per switching event).

## Application 3: Modem UART Level Shifting (On Daughter Card)

Four BSS138 instances on the modem daughter card shift between 3.3V (main MCU) and 1.8V (EG800Q-EU modem):

| Signal | Direction | Low Side | High Side |
|--------|-----------|----------|-----------|
| TXD | MCU → Modem | 1.8V (modem) | 3.3V (MCU) |
| RXD | Modem → MCU | 1.8V (modem) | 3.3V (MCU) |
| PWRKEY | MCU → Modem | 1.8V (modem) | 3.3V (MCU) |
| RESET_N | MCU → Modem | 1.8V (modem) | 3.3V (MCU) |

**UART speed:** 115200 baud. At this speed, the BSS138 level shifter has no difficulty — the bit period (8.7us) is orders of magnitude longer than the FET switching time (~10ns).

## Power Consumption

The BSS138 itself consumes effectively zero power:
- Gate leakage: ~1nA (negligible)
- When OFF: Ids leakage ~1nA
- When ON as inverter: only conducts the SI2301 gate charge current during switching events (~nA average)
- When ON as level shifter: only conducts during I2C transitions (~nA average)

The pull-up resistors consume standing current when the bus is LOW:
- 4.7k to 3.3V: 3.3V / 4.7k = 0.7mA per resistor when bus is LOW
- In sleep (bus idle HIGH): 0mA

## Known Issues / Errata

1. **Vgs(th) variability at 1.8V gate drive.** The 0.5V-1.5V threshold range means that at 1.8V gate voltage, some units have only 0.3V overdrive. For I2C level shifting, this increases Rds(on) and slows transitions. At 3.3V gate drive (inverter application), this is not a concern. **Mitigation:** If I2C Fast Mode reliability is needed, test with worst-case Vgs(th) devices or select a lower-threshold MOSFET (e.g., NXP 2N7002P with 0.35V-1.0V Vgs(th)).

2. **Parasitic capacitance impact on high-speed signals.** Ciss = 30pF adds to bus capacitance on both sides of the level shifter. For I2C at 100-400kHz this is manageable. For SPI or other high-speed interfaces, BSS138 level shifting is NOT suitable — use a dedicated level shifter IC or operate both sides at the same voltage.

3. **Body diode conduction.** During the level-shifting transition when the high side pulls low, the body diode conducts briefly. This is by design (part of the AN10441 operating principle) and is not a problem. However, the body diode's ~1.1V forward voltage means the low side is pulled to ~1.1V (not 0V) during the initial transition — the FET then turns on fully and pulls it to near 0V. This creates a brief voltage glitch visible on an oscilloscope but not functionally problematic for I2C.

4. **Not suitable for high-current switching.** Id max = 200mA. The BSS138 is for signal-level applications only. For power switching, use the SI2301CDS or similar power MOSFET.

5. **Multiple manufacturers.** Ensure the LCSC part C49139 is the onsemi BSS138, not a different manufacturer's variant. While electrically similar, pinout confirmation against the specific manufacturer's datasheet is essential.

6. **SOT-23 pinout: G-S-D (1-2-3).** This differs from some other SOT-23 N-channel MOSFETs that use G-D-S pinout. Double-check the footprint orientation in the PCB library matches the onsemi BSS138 pinout.
