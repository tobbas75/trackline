    STEP 1 — Add db:types to portal, camera-trap-dashboard, trap-monitor package.json:

    All three use the same Supabase project. fire-app already has db:types.
    Add this script to portal/package.json, camera-trap-dashboard/package.json,
    and trap-monitor/frontend/package.json scripts sections:

    ```json
    "db:types": "supabase gen types typescript --project-id $SUPABASE_PROJECT_ID > src/lib/supabase/database.types.ts"
    ```

    Add one entry to the scripts section of the root package.json:

    ```json
    "db:types": "pnpm --filter portal run db:types && pnpm --filter camera-trap-dashboard run db:types && pnpm --filter fire-app run db:types && pnpm --filter trap-monitor-dashboard run db:types"
    ```

    Preserve all existing scripts. Only add the new db:types entry.

    NOTE: This script requires SUPABASE_PROJECT_ID to be set in the shell environment.
    It will fail gracefully with a supabase CLI error if the variable is unset — the command
    does not need a guard here; developers are expected to have the variable set when running
    type generation. Document this requirement in the script with a comment if package.json
    supports it (it does not — so no comment needed).
