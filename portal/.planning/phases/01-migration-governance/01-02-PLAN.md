  <action>
    Create scripts/db-check.cjs. The complete implementation is specified in RESEARCH.md "Code Examples" section. Implement it exactly as specified, making no changes to the logic or check patterns.

    The file must:
    1. Start with `#!/usr/bin/env node` shebang and `'use strict';`
    2. Use only `require('fs')` and `require('path')` — no npm dependencies
    3. Implement `stripComments(sql)` that removes both `/* ... */` block comments and `-- ...` line comments
    4. Implement `normalise(sql)` that calls `stripComments` then `.replace(/\s+/g, ' ').toLowerCase()`
    5. Define `CHECKS` array with exactly these 8 check objects (IDs must match PROTECTED_SURFACES.md):
       - `drop_check_app_access`: pattern `/drop\s+function.*portal\.check_app_access/`
       - `signature_change_check_app_access`: detects `CREATE OR REPLACE FUNCTION portal.check_app_access(...)` with a signature other than `target_app_id text` (extract argument list from regex match, normalise whitespace, compare against exact string `'target_app_id text'`)
       - `drop_portal_app_access`: pattern `/drop\s+table.*portal\.app_access/`
       - `drop_portal_profiles`: pattern `/drop\s+table.*portal\.profiles/`
       - `drop_portal_apps`: pattern `/drop\s+table.*portal\.apps/`
       - `drop_on_auth_user_created`: pattern `/drop\s+trigger.*on_auth_user_created/`
       - `drop_column_profiles`: pattern `/alter\s+table\s+portal\.profiles\s+drop\s+column/`
       - `drop_column_app_access`: pattern `/alter\s+table\s+portal\.app_access\s+drop\s+column/`
    6. Main loop: read each file from `process.argv.slice(2)`, call `normalise()`, run all checks, collect violations
    7. Print per-file: `db-check OK: {filename}` on success, or `db-check FAILED: {filename}` + per-violation `  [{id}] {message}` lines on failure
    8. After all files: if violations > 0, print `{N} violation(s) found. Migration blocked.` and `process.exit(1)`; else print `All checks passed.` and `process.exit(0)`
    9. Exit code 2 for: no files passed (print usage), file not found (print error)

    Error messages must be descriptive — include which downstream apps are affected. Use the message strings from RESEARCH.md CHECKS array verbatim.
