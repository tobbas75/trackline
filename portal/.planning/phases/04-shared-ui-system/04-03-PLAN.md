**Step 2 — Migrate portal/src/app/globals.css:**

Replace the entire file with a minimal version that imports tokens.css and keeps only the
font family override (since fonts are injected per-app via next/font CSS variables, not in tokens.css):

```css
@import "tailwindcss";
@import "../../packages/ui/tokens.css";

@source "../../packages/ui/src";

/* App-level font binding — injected by next/font at runtime */
@theme inline {
  --font-sans: var(--font-poppins);
  --font-display: var(--font-dm-serif);
}

body {
  background: var(--background);
  color: var(--foreground);
  font-family: var(--font-sans), system-ui, sans-serif;
}

html {
  scroll-behavior: smooth;
}
```

NOTE: The grain, img-zoom, dust-line, animate-fade-up, and animation-delay utilities now come from
tokens.css, so they do NOT need to be repeated in portal's globals.css.

Write a practical reference covering:

# Trackline UI Rules

## Design Principles
- Earthy, minimal, clean — Between23 aesthetic
- DM Serif Display for headings, Poppins for body
- Whitespace is generous, typography is tight
- Rounded corners: rounded-sm (2px) only — no rounded-lg or rounded-xl

## Colour System
Brief table: token name | CSS var | hex | use case
(red-dust, ochre, eucalypt, sky, stone scale, semantic aliases)

## Typography
- Display headings: font-[family-name:var(--font-dm-serif)] or font-display
- Body: default (Poppins via --font-sans)
- Section labels: text-xs font-semibold tracking-widest uppercase text-red-dust
- Sizes: text-sm body, text-xl–text-3xl headings

## Layout
- Max content width: max-w-5xl (dashboard) or max-w-6xl (marketing)
- Container padding: px-6 py-4 (header), px-6 py-12 (main content)
- Card grid: grid gap-6 sm:grid-cols-2 lg:grid-cols-3
- Breakpoints: sm (640px), md (768px), lg (1024px)

## Components (from @trackline/ui)
Button: primary | secondary | ghost — sm | md
Card / CardHeader / CardBody — hover prop for interactive cards
Badge: default | primary | ochre | eucalypt | sky
Avatar: sm | md | lg — initials fallback

## Conventions
- Server Components by default — "use client" only for event handlers/hooks
- Named exports only (except Next.js page/layout)
- Icons: lucide-react only
- No inline styles — Tailwind classes only
- cn() from @trackline/ui for conditional classes
